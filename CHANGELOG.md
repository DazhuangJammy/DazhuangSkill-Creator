# 更新日志

## v1.5.2 - 2026-04-11

- 新建 skill 时，“要不要开记忆”现在是必做检查，不能悄悄跳过。
- 如果你用 `memory_mode=auto`，但没写 `--intent`，而且系统先判断成 `off`，会先停下来提醒你补充信息。
- 新建流程里会更清楚地打印“记忆判型结果”，让你一眼看到这次是怎么判的。
- 文档也补上了这条硬要求：就算最后选 `off`，也要先判断并说清楚理由。
- 补了回归测试，防止以后又出现“没想清楚就把记忆关掉”。

## v1.5.1 - 2026-04-11

- 修了一个“容易误关记忆”的坑。
- 现在只要最后是 `memory_mode=off`（命令行或配置文件），但系统判断这个 skill 明明更该开记忆，就会先拦住并提醒你。
- 只有你明确加 `--force-memory-off`，才会真的强制关闭记忆。
- 补了回归测试，防止这个坑以后再出现。

## v1.5.0 - 2026-04-11

- 这是一次大更新：skill 现在更会“长记性”了。
- 新建 skill 时，系统会先判断记忆模式；你也可以手动选。
- 记忆模式有 4 种：`off`、`adaptive`、`lessons`、`auto`（默认）。
- 开记忆后，会有 3 个“小本本”：状态本（`memory-state.json`）、过程本（`memory-events.jsonl`）、经验本（`memory-lessons.md`）。
- 同一个坑反复出现，会先写进经验本；后面一直做对，这条经验会升级成 `SKILL.md` 里的硬规则。
- 新增防回归测试，避免“已经退休的经验”马上被重新激活。
- `quick_validate.py` 更严格了：记忆模式少关键步骤会直接提醒。

## v1.4.0 - 2026-04-07

- 这版重点是：Mac 和 Windows 都能更稳地跑。
- 修了编码相关坑，避免在 Windows 上出现中文/文件读取异常。
- 评测脚本也改成了跨平台更稳的方式，不再依赖 Unix 特有行为。
- 文档里的命令写法统一成跨平台思路（Windows 可用 `py -3`）。

## v1.3.1 - 2026-04-06

- README 里把安装方式说得更清楚：推荐用 `git clone`，不要手动复制文件夹。
- 这版主要是把“安装与更新”这条路走顺。

## v1.3.0 - 2026-04-06

- 就算更新失败，也不会卡住你当前任务（不阻断主流程）。
